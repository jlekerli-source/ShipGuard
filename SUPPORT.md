# Support

ShipGuard support is public and issue-driven. The project is built for maintainers who want Codex and similar agents to plan, validate, and hand off work with evidence.

## Use GitHub Issues For

- ShipGuard CLI bugs.
- Broken plugin install or Codex status behavior.
- Incorrect report output, noisy findings, missing proof guidance, or unsafe shareability behavior.
- Reproducible documentation gaps.
- Public fixture or eval failures.

## Use GitHub Discussions For

- Adoption questions.
- Workflow design questions.
- Ideas that are not yet concrete bugs or proposals.
- Comparing ShipGuard with other agent-workflow tools.

## Do Not Post

- Secrets, API keys, bearer tokens, Apple team IDs, App Store Connect credentials, customer data, or private screenshots.
- Private app source code unless you own it and intentionally want it public.
- Token-bearing Devspace URLs or local report paths that reveal private machine details.

Use `shipguard ios redact` before sharing generated reports. For security-sensitive problems, follow `SECURITY.md` instead of opening a public issue with private details.

## What A Useful Report Includes

- ShipGuard version.
- Command run.
- Operating system and Codex/plugin context when relevant.
- Minimal public fixture or redacted report.
- Expected behavior.
- Actual behavior.
- Validation command or proof artifact.
