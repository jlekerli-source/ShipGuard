# Roadmap

This roadmap keeps the repo useful without pretending it is a finished framework.

## Now

- Tighten the root `AGENTS.md` template for iOS projects with alarms, notifications, widgets, payments, and release gates.
- Add small Codex skills that can be copied into other repositories.
- Keep the evaluation suite realistic and tied to product-maintenance risk.

## Next

- Add example before/after Codex prompts for a bug fix, UI polish pass, release pass, and review pass.
- Add a `scripts/validate_workflow_bundle.sh` command that checks docs, executable scripts, and required files.
- Add a sample issue-to-plan-to-validation transcript with private app details removed.

## Later

- Publish a small collection of anonymized maintainer workflows from real Ringly work.
- Add GitHub Actions for markdown linting and shell syntax checks.
- Build a lightweight scorecard for evaluating whether an agent run produced usable maintenance evidence.
