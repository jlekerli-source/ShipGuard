# Code Of Conduct

ShipGuard is an open-source project for making AI-assisted software work more reviewable, testable, and honest. The community standard is simple: focus on evidence, keep discussion respectful, and do not pressure maintainers or contributors into unsafe disclosure or unsafe automation.

## Expected Behavior

- Be direct about bugs, risks, and weak assumptions without attacking people.
- Share reproduction steps, commands, fixtures, screenshots, or redacted reports when possible.
- Respect project boundaries around private apps, credentials, screenshots, customer data, and unreleased product details.
- Accept maintainer decisions about scope, validation, release timing, and security handling.
- Keep public examples generic, redacted, and reusable.

## Unacceptable Behavior

- Harassment, insults, threats, sexualized language, or discriminatory comments.
- Posting credentials, private app source, customer data, local paths, Apple account identifiers, or screenshots without permission.
- Asking contributors to scan, probe, or modify systems they do not own.
- Repeatedly reopening the same argument after a maintainer has given a clear decision.
- Using ShipGuard issue threads for spam, promotion, or unrelated support.

## Enforcement

Maintainers may edit, hide, lock, or delete comments; close issues; block users; or restrict repository access when behavior makes the project unsafe or unproductive. Security-sensitive content may be removed without debate.

Report conduct problems through a GitHub issue only when no private details are involved. If private details are involved, follow `SECURITY.md` instead.
